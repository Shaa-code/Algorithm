### 왜 못 풀었는가?

tmp = null;을 넣는 아이디어를 스스로 발생해낼수가 없다.

잘쓴 사람의 코드를 보고 필연적으로 그렇게 풀어나갈수 밖에 없는 이유를 찾아가자.

### 잘 쓴 사람의 코드

```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.LinkedList;
import java.util.Queue;
import java.util.StringTokenizer;

class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        Main T = new Main();
        StringTokenizer st = new StringTokenizer(br.readLine());
        int n = Integer.parseInt(st.nextToken());
        int k = Integer.parseInt(st.nextToken());
        int[] arr = new int[n];
        st = new StringTokenizer(br.readLine());
        for(int i = 0; i < n ; i++){
            arr[i] = Integer.parseInt(st.nextToken());
        }
        System.out.println(T.solution(n, k, arr));
    }

    public int solution(int n, int k, int[] arr) {
        Queue<Patient> q = new LinkedList<>();
        for (int i = 0; i < n; i++) q.offer(new Patient(i, arr[i]));
        int cnt = 0;
        while (!q.isEmpty()) {
            Patient tmp = q.poll();
            for (Patient p : q) {
                if (p.priority > tmp.priority) {
                    q.offer(tmp);
                    tmp = null;
                    break;
                }
            }
            if (tmp != null) {
                cnt++;
                if (tmp.num == k) return cnt;
            }
        }
        return cnt;
    }
}
class Patient{
    int num;
    int priority;
    Patient(int num, int priority){
        this.num = num;
        this.priority = priority;
    }
}
```

### 필연적인 발상의 순서

1. q.offer(tmp) 이후 break를 하면 for문이 끝나고 아래 내용이 실행된다.

```java
while(!q.isEmpty()){
    Patient tmp = q.poll();
    for(Patient p : q){
        if(p.getPriority() > tmp.getPriority()){
            q.offer(tmp);
            break;
        }
    }
        cnt++;
        if (tmp.getNum() == k) return cnt;
    }
}
```

하지만 밑에 cnt++;부터는 그냥 실행되어서는 안된다. 어떤 조건이 있어야한다.

2. 어떤 조건일지 생각을 해보자.

```java
while(!q.isEmpty()){
    Patient tmp = q.poll();
    for(Patient p : q){
        if(p.getPriority() > tmp.getPriority()){
            q.offer(tmp);
            break;
        }
    }
    if(){
        cnt++;
        if (tmp.getNum() == k) return cnt;
    }
}
```

위험도가 더 큰 환자가 없다면, for문을 돌더라도 if문은 작동하지 않는다.

즉, if문을 실행시키지 않았을때, 남아있는 어떤 값으로 기준을 세우면 된다.

3. flag를 사용해서 만들어보았다.

```java
while (!q.isEmpty()) {
    boolean flag = false;
    Patient tmp = q.poll();
    for (Patient p : q) {
        if (p.getPriority() > tmp.getPriority()) {
            q.offer(tmp);
            flag = true;
            break;
        }
    }
    if (flag == false) {
        cnt++;
        if (tmp.getNum() == k) return cnt;
    }
}
```

4. 하지만 잘 생각해보면, flag의 역할을 tmp가 할수 있다.


변수의 쓰임새를 잘 확인하는게 정말 중요하다.

### 배운점

1. 접근하는 방법을 배웠다.

2. 딱히 배웠다기보다, 다른점이 있다. Patient의 멤버변수를 private으로 선언하지 않아서 바로 접근할 수 있게 해놨다.

난 개발하던 습관이 있어서 JavaBean규약을 적용했다. 어떤게 코딩테스트 시에 이점이 될지 모르겠다.
